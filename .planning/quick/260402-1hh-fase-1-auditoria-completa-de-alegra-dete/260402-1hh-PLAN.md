
    2. POST /api/auditoria/aprobar-limpieza (require_admin):
       - Recibe { confirmado: bool, excluir_ids: [int] }
       - Si confirmado == False: retorna {"status": "plan_sin_ejecutar", "mensaje": "Aprobacion no confirmada. No se ejecuto ninguna accion.", "excluir_ids": [...]}. NO escribe en MongoDB.
       - Si confirmado == True: inserta en coleccion "auditoria_aprobaciones" un doc con:
         * aprobado_por: current_user["username"]
         * timestamp: datetime.now(timezone.utc)
         * excluir_ids: request.excluir_ids
         * status: "aprobado_pendiente_ejecucion"
       - Retorna {"status": "aprobado", "doc_id": str(inserted_id), "mensaje": "Aprobacion registrada. La ejecucion se realizara en Fase 2."}
       - Necesita import de database.py: from database import db

    3. POST /api/auditoria/anular-bill-duplicada (require_admin):
       - Recibe { bill_id_a_anular: int, bill_id_a_mantener: int }
       - Paso 1: Verificar que bill_id_a_anular existe en Alegra via GET /bills/{bill_id_a_anular}
         Si no existe o status >= 400 → HTTPException 404 "Bill {id} no encontrada en Alegra"
       - Paso 2: Verificar que bill_id_a_mantener existe (misma logica)
       - Paso 3: Registrar evento en MongoDB coleccion "roddos_events":
         * event_type: "bill_duplicada_anulada"
         * agent: "auditoria"
         * payload: { bill_anulada: bill_id_a_anular, bill_mantenida: bill_id_a_mantener }
         * timestamp: datetime.now(timezone.utc)
         * source: "routers/auditoria.py"
       - Paso 4: Ejecutar DELETE /bills/{bill_id_a_anular} via httpx directo
         IMPORTANTE: Usar httpx.AsyncClient directo (patron existente en eliminar-journal),
         NO AlegraService.request() — porque validate_delete_protection() bloquea DELETE en bills.
         Bills de Auteco SI se pueden anular desde auditoria con aprobacion explicita.
       - Retorna { "anulado": True, "bill_id": bill_id_a_anular, "bill_mantenida": bill_id_a_mantener, "evento_registrado": True }

    REGLAS INAMOVIBLES:
    - validate_delete_protection() NO se toca — bills siguen protegidas para agentes automaticos
    - Auditoria bypasea la proteccion via httpx directo (mismo patron que eliminar-journal existente)
    - Toda escritura en Alegra registrada en roddos_events
