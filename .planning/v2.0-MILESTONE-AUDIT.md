## Critical Blockers (gaps_found)

### Blocker 1: Phase 4 Files Missing from `main` HEAD

**Root cause:** `agent_prompts.py`, `agent_router.py`, `portfolio_pipeline.py`, and `test_phase4_agents.py` were committed on feature branch `claude/nostalgic-boyd` (commits: `fef5ed7`, `dcbb2e2`, `927dca0`) but were never merged into `main`. These files do not exist on disk at `main` HEAD.

**Impact:**
- 9 requirements unsatisfied: AGT-01, AGT-02, AGT-03, AGT-04, SCH-01, SCH-02, SCH-04, TST-04, TST-06
- Production `ai_chat.py` still uses `is_cfo_query()` / `CFO_KEYWORDS` (keyword routing, not confidence-based)
- `cfo_agent.py` retains old routing logic that Phase 4 was supposed to replace
- Daily portfolio pipeline (23:30 job) never runs — CFO has no pre-computed summaries to read
- 2 E2E flows completely broken

**Fix:** Cherry-pick Phase 4 commits from `claude/nostalgic-boyd` to `main`:
```bash
git cherry-pick fef5ed7  # agent_prompts.py
git cherry-pick dcbb2e2  # agent_router.py
git cherry-pick 927dca0  # portfolio_pipeline.py + test_phase4_agents.py
```
Then re-verify Phase 4 on `main`.

### Blocker 2: CI Pipeline Will Break on Next Push

**Root cause:** `.github/workflows/ci.yml` line 66 lists `test_phase4_agents.py` in the pytest command. This file does not exist in `main`. The `pytest-build24` job will fail with `ERROR: not found: test_phase4_agents.py` on the first push.

**Impact:** GIT-01 partial — CI is not functional as defined. Every push to main will trigger a failing build.

**Fix:** Either cherry-pick Phase 4 commits (resolves both blockers), or temporarily remove `test_phase4_agents.py` from the pytest command in `ci.yml` until Phase 4 is merged.

### Blocker 3: Phase 3 Unverified

**Root cause:** Phase 3 completed (3 SUMMARY.md files, all 3 plans executed) but no VERIFICATION.md was created during `execute-phase`.

**Impact:** 10 requirements (MDB-01..09, TST-03) classified as `partial` — SUMMARY evidence exists but no formal verification.
