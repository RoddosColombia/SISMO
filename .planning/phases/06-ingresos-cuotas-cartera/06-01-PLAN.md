**Problema raiz:** `from routers.cartera import registrar_pago_cartera` en cada test dispara
`routers/__init__.py` que importa `auth.py` → `security_service.py` → `import qrcode` (no instalado
en CI). Esto hace fallar T1-T6 con `ModuleNotFoundError: No module named 'qrcode'`.

**Fix:** Reemplazar el import dentro de cada test function por un import directo al modulo
sin pasar por el paquete `routers`:

```python
import importlib
import sys

def _import_cartera():
    """Import cartera module bypassing routers/__init__.py chain."""
    import importlib.util, pathlib
    spec = importlib.util.spec_from_file_location(
        "cartera_module",
        pathlib.Path(__file__).parent.parent / "routers" / "cartera.py",
    )
    mod = importlib.util.module_from_spec(spec)
    # Patch db before exec so module-level imports resolve
    sys.modules.setdefault("database", MagicMock())
    spec.loader.exec_module(mod)
    return mod
```

En la practica, la forma mas limpia compatible con como ya estan escritos los tests es agregar
al inicio del archivo (despues de los imports existentes) un bloque que parchea `qrcode` antes

```python
# ── ISOLATION: stub qrcode before any router import chain ──
import sys
from unittest.mock import MagicMock
```

```python
@pytest.mark.asyncio
async def test_t7_duplicado_detectado(mock_db, mock_user, loanbook_con_cuotas_pendientes):

    T7: Si ya existe un pago para loanbook_id + cuota_numero + fecha_pago,
    el endpoint debe retornar HTTP 409 con mensaje que contiene "duplicado".

    ESTADO: RED — cartera.py aun no tiene este guard. Plan 06-02 lo implementa.

    from routers.cartera import registrar_pago_cartera, RegistrarPagoRequest
    from fastapi import HTTPException

    # Simular que ya existe un pago previo para esta cuota
    mock_db.cartera_pagos.find_one = AsyncMock(return_value={
        "id": "PAGO-JE-PREV-0001",
        "loanbook_id": "LB-2026-0042",

        "fecha_pago": "2026-03-22",
```
