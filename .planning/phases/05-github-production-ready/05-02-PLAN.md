    ```yaml
      name: BUILD 24 Test Suite
      runs-on: ubuntu-latest
      needs: backend-check
      steps:

        - uses: actions/setup-python@v5

            python -m pytest tests/test_permissions.py tests/test_event_bus.py tests/test_mongodb_init.py tests/test_phase4_agents.py tests/test_smoke_build24.py -v --tb=short
    ```

    **ADD step to backend-check job: anti-pending-status** (per GIT-01):
    Add this step AFTER the "No emergentintegrations" step:
    ```yaml
      - name: No pending status markers
        run: |
          if grep -rn "status.*['\"]pending['\"]" backend/services/ backend/routers/ --include="*.py" | grep -v "test_" | grep -v "#.*pending" | grep -v "pendiente"; then
            echo "ERROR: found 'pending' status marker — use 'processed' or 'pendiente' instead"
            exit 1
          fi
          echo "OK - no pending status markers"
    ```

    Note: The anti-pending check looks for English "pending" (the old bus pattern) in services/ and routers/ Python files, excluding test files and comments. Spanish "pendiente" (used for cuota estado) is allowed.

    **ADD Job: smoke-post-deploy** (per D-03, D-04, D-05, GIT-02):
    ```yaml
    smoke-post-deploy:
      name: Post-Deploy Smoke Test
      runs-on: ubuntu-latest
      needs: pytest-build24
      if: github.ref == 'refs/heads/main' && github.event_name == 'push'
      steps:
        - name: Wait for Render deploy
          run: sleep 90
        - name: Smoke test production
          run: |
            RESPONSE=$(curl -sf https://sismo-backend-40ca.onrender.com/api/health/smoke)
            echo "$RESPONSE" | python3 -m json.tool

            STATUS=$(echo "$RESPONSE" | python3 -c "import sys,json; print(json.load(sys.stdin)['status'])")
            COLLS=$(echo "$RESPONSE" | python3 -c "import sys,json; print(json.load(sys.stdin)['collections_count'])")
            BUS=$(echo "$RESPONSE" | python3 -c "import sys,json; print(json.load(sys.stdin).get('bus_status','unknown'))")

            echo "Status: $STATUS"
            echo "Collections: $COLLS"
            echo "Bus: $BUS"

            if [ "$STATUS" = "critico" ]; then
              echo "FAIL: smoke test returned critico"
              exit 1
            fi

            if [ "$COLLS" -lt 30 ] 2>/dev/null; then
              echo "WARN: collections_count=$COLLS (expected >= 30)"
            fi

            echo "Smoke test passed"
    ```

    The final ci.yml should have 4 jobs total: backend-check, frontend-check, pytest-build24, smoke-post-deploy.
