async def sincronizar_journals_recientes() -> int:
    Hace GET /journals paginado (offset 0, limit 30) y guarda/actualiza cada journal
    en la colección alegra_journals_cache de MongoDB.
    
    Schema de cada documento en alegra_journals_cache:
      alegra_id: str (ID de Alegra, índice único)
      fecha: str (YYYY-MM-DD)
      observations: str
      total_debito: float (suma de todos los entries con debit > 0)
      entries: list[dict] con {account_id, debit, credit}
      synced_at: str (ISO UTC)
      estado: str ("open" | "closed" | desconocido)
    
    Upsert por alegra_id — nunca duplicar.
    Retorna cantidad de journals nuevos o actualizados.
    Manejo de errores: si Alegra da 503/429 → loguear y retornar 0 sin lanzar excepción.
    Si is_demo_mode() → retornar 0 inmediatamente.

PASO 4 — GUARD ANTI-DUPLICADO EN bank_reconciliation.py
========================================================

En backend/services/bank_reconciliation.py, en el método _crear_journal_en_alegra(),
ANTES de la línea que hace el POST a Alegra (result = await alegra.request_with_verify(...)),
agregar esta verificación:

    # GUARD: verificar si ya existe en alegra_journals_cache (Alegra→SISMO sync)
    existe_en_alegra = await self._journal_ya_en_cache(movimiento.fecha, movimiento.monto)
    if existe_en_alegra:
        journal_id_existente = existe_en_alegra.get("alegra_id", "desconocido")
        self.logger.warning(
            f"[GUARD] Movimiento ya existe en Alegra (cache): "
            f"fecha={movimiento.fecha} monto={movimiento.monto} "
            f"journal_id={journal_id_existente} — saltando"
        )
        return False, None, f"duplicado_en_alegra:{journal_id_existente}"

Implementar _journal_ya_en_cache() como método de la clase BankReconciliationService:
    async def _journal_ya_en_cache(self, fecha: str, monto: float) -> dict | None:
        Busca en alegra_journals_cache un documento donde:
          - fecha == fecha exacta (YYYY-MM-DD)
          - abs(total_debito - monto) <= 1.0  (tolerancia de $1 COP por redondeos)
        Retorna el documento si existe, None si no.
        Si la consulta falla por cualquier motivo → retornar None (fail-open: si el cache
        no está disponible, se permite la creación para no bloquear la operación).
